實際文件範例 - 「用戶登入系統」專案

📋 requirements.md

# 用戶登入系統需求文件

## 功能需求 (EARS 格式)

### REQ-001: 用戶登入
**When** 用戶在登入頁面輸入有效帳號密碼
**The system shall** 驗證憑證並導向主頁面
**Within** 3秒內完成驗證

### REQ-002: 錯誤處理  
**When** 用戶輸入無效憑證
**The system shall** 顯示錯誤訊息
**And** 保持在登入頁面

### REQ-003: 記住登入
**When** 用戶勾選「記住我」選項
**The system shall** 儲存登入狀態
**For** 30天內免重新登入

## 非功能需求

- **效能**: 登入響應時間 < 3秒
- **安全**: 密碼需雜湊加密儲存
- **可用性**: 支援鍵盤 Tab 導航

🏗️ design.md

# 用戶登入系統設計文件

## 系統架構

### 前端層 (React)
- LoginForm.tsx
- AuthContext.tsx  
- ProtectedRoute.tsx

### API 層 (Node.js/Express)
- POST /api/auth/login
- POST /api/auth/logout
- GET /api/auth/verify

### 資料層 (MySQL)
- users 表格
- sessions 表格

## 元件設計

### LoginForm 元件
```typescript
interface LoginFormProps {
  onSubmit: (credentials) => void;
  isLoading: boolean;
  error?: string;
}
```

## 資料流程
1. 用戶輸入 → LoginForm
2. 表單驗證 → AuthContext  
3. API 呼叫 → Backend
4. JWT Token → LocalStorage
5. 路由重導 → Dashboard

## 安全考量
- HTTPS 加密傳輸
- JWT Token 過期機制
- CSRF 保護

✅ tasks.md

# 用戶登入系統任務清單

## 第一階段：基礎建設 (3天)

### [x] T1.1 專案環境建立 
- 建立 React + Express 專案
- 配置 TypeScript 設定
- **責任人**: 前端工程師
- **完成日期**: 2025-07-15

### [ ] T1.2 資料庫設計 (高優先級)
- 設計 users 表格結構  
- 建立 sessions 表格
- **責任人**: 後端工程師
- **預計時間**: 1天

### [ ] T1.3 API 架構設計 (高優先級)
- 設計認證 API 端點
- 定義請求/回應格式
- **責任人**: 全端工程師
- **預計時間**: 1天

## 第二階段：核心功能 (5天)

### [ ] T2.1 登入表單製作 (中優先級)
- 建立 LoginForm 元件
- 表單驗證邏輯
- **責任人**: 前端工程師

### [ ] T2.2 後端認證 API (高優先級)  
- 實作 /api/auth/login
- JWT Token 生成
- **責任人**: 後端工程師

### [ ] T2.3 前端狀態管理 (中優先級)
- AuthContext 實作
- 登入狀態同步
- **責任人**: 前端工程師
